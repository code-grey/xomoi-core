2. SECURITY: THE HMAC-LITE HANDSHAKE
-----------------------------------------------------------------------
Xomoi replaces memory-intensive mTLS/JWT with HMAC-Lite for edge nodes.

SIGNATURE = HMAC-SHA256(PAYLOAD + TIMESTAMP, DEVICE_TOKEN)

Step-by-Step Flow:
1. Sensor: Packs Protobuf TelemetryBatch.
2. Sensor: Signs Payload + current Unix Timestamp using Device_Token.
3. Node: Verifies Signature within a 60-second drift window.
4. Node: If valid, persists to in-memory Hot State.

3. ADAPTIVE PRUNING (THE XOMOI-CTL TRANSPILER)
-----------------------------------------------------------------------
To protect the tiny RAM of 8-bit/32-bit sensors, Xomoi uses a
"Holey Header" generation technique.

MASTER PROTO (ALL FIELDS)        LITE SDK HEADER (PRUNED)
+------------------------+      +------------------------+
| tag_id = 1             |      | tag_id = 1             |
| f_val  = 2             |      | f_val  = 2             |
| i_val  = 3             |      | i_val  = 3             |
| raw_val = 8 (IMAGES)   |----->| // [PRUNED FOR RAM]    |
| json_val = 9 (AI)      |----->| // [PRUNED FOR RAM]    |
+------------------------+      +------------------------+
(Go Backend Sees All)           (Sensor Allocates ~8 Bytes)

4. FLASH-WEAR PROTECTION (SD-CARD LONGEVITY)
-----------------------------------------------------------------------
Xomoi preserves the life of edge storage (SD/eMMC) by minimizing writes.

- REAL-TIME DATA: Handled entirely in-memory (Go sync.Map).
- SNAPSHOT CYCLE: Every 5 minutes, Xomoi flushes the Hot State to
  SQLite WAL (Write-Ahead Log) in a single atomic transaction.
- JANITOR: Background aggregation of telemetry into 1m, 5m, and 1h
  buckets to ensure O(1) charting performance.
