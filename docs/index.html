Embedded Mochi-MQTT

Xomoi embeds a fully compliant MQTT v5 broker directly into the binary. By utilizing a 256-shard FNV-1a Hash Map, it completely eliminates Mutex lock contention, capable of routing hundreds of thousands of messages per second.

internal/broker/mochi.go

Lossless Zstd TSDB

Traditional DBs destroy SD cards. Xomoi streams packets into a Go Ring Buffer, natively compresses JSON payloads using the Zstandard algorithm, and executes asynchronous Bulk Inserts into a WAL-enabled SQLite database.

internal/state/ring_buffer.go

The Brutal Truth

Xomoi isn't a silver bullet. We built it to solve a very specific problem: deploying mission-critical IoT infrastructure on fragile, low-power edge devices without the nightmare of managing 5 different Docker containers.

Where Xomoi Dominates

Vs KubeEdge & Docker: You don't need a heavy Containerd daemon or a Kubernetes control plane. Xomoi runs as a single executable, radically reducing CPU/RAM overhead and eliminating moving parts that break in remote fields.

Vs Mosquitto + InfluxDB + Redis: Instead of piping data over TCP loops between a broker, a cache, and a database, Xomoi does it all in-memory. By natively compressing payloads with Zstd before hitting SQLite, we extend SD card lifespans by years.

Where Xomoi Fails

Massive Cloud Data Centers: Xomoi is strictly an edge-first, offline-first node. If you are building a central cloud cluster handling millions of concurrent vehicles globally, use EMQX or Kafka.

Independent Microservice Scaling: If your architecture relies on independently scaling the database layer from the broker layer across multiple servers, Xomoi's hyper-converged monolith approach will fight you.

The Sovereign Roadmap

PHASE 10.1

Hardware RTC Deep Sleep

Command edge endpoints (like ESP32s) to enter deep sleep via Retained MQTT RPCs. Xomoi will act as a buffer, waking devices on a precise schedule to transmit data, preserving lithium batteries for years.

PHASE 11.1

The Immortal Gossip Mesh

Distributed High-Availability across multiple Xomoi nodes on a local LAN. Using UDP-based epidemic routing (Hashicorp Memberlist) and CRDTs to achieve Split-Brain immunity during network partitions.

PHASE 13.0

Silo Edge Orchestration

Transforming Xomoi into a KubeEdge competitor. By refactoring our custom silo-core engine, Xomoi will leverage pure Linux cgroups and namespaces to run untrusted 3rd-party AI plugins inside secure, memory-limited rootfs environments—without ever installing Docker.
